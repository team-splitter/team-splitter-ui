## Overview

Team Splitter Admin UI - a Create React App (react-scripts 5) + TypeScript SPA for managing pickup-sports polls, players, game schedules, and auto-generated team splits. It is a thin admin frontend over a separate backend REST API.

## Commands

```sh
npm install            # install deps (yarn.lock is also committed; npm ci is used in Docker)
npm start              # dev server at http://localhost:3000 (REACT_APP_ENV=local via .env.development)
npm run build          # production build to /build
npm test               # jest in watch mode (react-scripts/jest)
npm test -- --watchAll=false <pattern>   # run tests once, optionally filtered by file/name
```

There is no separate lint script; ESLint runs through react-scripts using the `react-app` + `react-app/jest` config in `package.json`. There are currently no test files in `src`.

## Required before the app will build or run: `amplifyconfiguration.json`

`src/index.tsx` imports `./amplifyconfiguration.json`, but that file is **gitignored** (see the amplify block in `.gitignore`). It must be generated by pulling the Amplify environment before `npm start`/`npm run build` will succeed:

```sh
amplify pull            # or `amplify env checkout <env>` then pull
```

See `README.md` for the full Amplify workflow (`amplify env`, `amplify push`, `amplify publish`).

## Architecture

**Auth (Auth0, not Cognito).** Although Amplify provisions a Cognito user pool in `amplify/backend/auth`, the running app authenticates with **Auth0** via `@auth0/auth0-react`. `src/index.tsx` wraps `<App>` in `<Auth0Provider>` using `domain`/`clientId` from config. `src/auth/PrivateRoute.tsx` gates routes: `routes` entries with `authRequired: true` are wrapped in `<PrivateRoute>`, which renders children only when `isAuthenticated`.

**Routing.** All routes are declared centrally in `src/routes.ts` as data (key/title/path/authRequired/component) and rendered by a `.map` in `src/App.tsx`. Add a page by adding an entry there; `navDisplay` controls whether it appears in the Navbar.

**Backend URL is discovered dynamically from Amplify, not from config.** `src/globalConfig.ts` `backendUrl()` returns `Amplify.getConfig().API?.REST?.teamsplitterapi.endpoint + "/api/v1"`. Service modules call `backendUrl()` at request time. The `backend.url` field in the `config/*.json` files is not used for API calls.

**Service layer.** `src/services/*Service.ts` are the only place that talks to the backend. Each exports plain async functions (e.g. `getPolls`, `addPollVote`) that build a URL from `backendUrl()` and call the shared HTTP client. Pages import these services directly - there is no Redux store in use (Redux Toolkit is a dependency but unused; state is local `useState`/`useEffect` per page).

**HTTP client (`src/commons/client/http.ts`).** Thin axios wrapper exporting `get/post/put/patch/del`. Two behaviors to know:
- `handleSuccess` unwraps a nested envelope: it returns `response.data.data` when present, otherwise `response.data`. Backend responses wrapped as `{ data: ... }` are automatically unpacked.
- `handleErrors` throws a normalized `HttpError` (`{ code, message }`), not the raw axios error. Callers `.catch` and read `err.message`.

**Configuration.** `src/services/configurationService.ts` holds a `Config` type and a `configSingleton`, selecting `config/local.json` or `config/prod.json` by env name. `src/services/context/ConfigurationContext.tsx` provides `withConfigurationProvider` / `withConfiguration` HOCs. Note the env selection is currently hardcoded/messy (the provider defaults to `'local'`; there is a `TODO` to replace the singleton with the context) - be careful when changing config wiring.

**UI.** MUI v5 (`@mui/material`, `@mui/x-data-grid`, `@mui/x-date-pickers` with the Moment adapter). Theme is defined inline in `App.tsx`. Pages live under `src/pages/<feature>/`; shared components under `src/components/`; API DTO types under `src/api/*.types.ts`.

**Domain data flow.** A `Poll` has votes (players who are going); `TeamSplitPage` generates team splits from the poll's votes; splits feed into games. `PollPage` orchestrates this with `refreshKey` counters passed to child pages to force re-fetch after mutations (vote change bumps `voteKey` → re-splits; split success bumps `splitKey` → refreshes games).

## Deployment

Production is a multi-stage Docker build (`Dockerfile`): builds with Node 16 then serves the static bundle from nginx. `core/nginx/nginx.conf` forces HTTPS, serves the SPA with `try_files ... /index.html`, and reverse-proxies `/api` to `http://backend:8080/`. Images are published to `ghcr.io/team-splitter/team-splitter-ui` via `dockerBuild.sh <version>` / `dockerPush.sh <version>`.
